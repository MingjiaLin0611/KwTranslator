# M3 功能规格

## 主要功能

KW Translator M3 将 M2 的单个用户词库升级为多个命名自定义词汇表。自定义词汇表是用户操作的最大单元，用户可以在 popup 内完成 M3 全部功能验证：新建、重命名、删除、启用、禁用、排序、导入和导出单个词汇表。

首次运行 M3 时，扩展初始化一个空的默认词汇表“我的词汇表”，默认启用。用户必须先选中一个词汇表，再在该词汇表内添加、编辑或删除词条。手动添加词条只暴露两个必填字段：原文和翻译。

示例：

`React uses API.`

如果启用的自定义词汇表包含 `React -> React 框架` 和 `API -> 自定义接口`，注入后：

`React(React 框架) uses API(自定义接口).`

## 次要功能

- popup 使用“词汇表列表 + 当前词汇表详情面板”布局。
- popup 暂时承载完整 M3 管理功能，后续里程碑再迁移到独立 options 页面。
- 内置词库只显示只读统计，不进入自定义词汇表列表。
- 多个自定义词汇表可以同时启用。
- 自定义词汇表优先级由列表顺序决定，用户通过上移和下移按钮调整。
- 同一词汇表内 normalized keyword 必须唯一，重复添加或编辑时提示并阻止保存。
- 不同词汇表可以包含相同 keyword，翻译时排序靠前的启用词汇表优先。
- 删除词汇表需要二次确认。
- 至少保留一个自定义词汇表，最后一个词汇表不能删除。
- 词汇表或词条变化后，当前活动页面应恢复旧注入并按最新词库重新注入。
- 单个词汇表可以导出为 JSON 文件，文件名使用词表名和导出日期。
- 导入同名词汇表时不覆盖本地词汇表，而是自动改名为副本。

## 非目标

- 真实 AI provider 集成。
- 整句或整页翻译。
- 云同步。
- 独立 options 管理页面。
- 内置词库编辑、删除、导出、排序或禁用。
- 兼容或迁移 M2 体验阶段保存的单用户词库数据。

## 角色分工

- `planner`：维护 M3 规格、验收标准和测试方向。
- `executor`：审查规格风险后按最小执行单元实现。
- `tester`：根据 M3 规格运行单元测试、覆盖率、编译、构建和 E2E。
- `overseer`：每个最小执行单元完成后检查文档、测试和代码是否同步。
