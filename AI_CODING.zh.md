# AI Coding 入口

KW Translator 的所有大型任务都必须使用四个分离角色：planner、executor、tester 和 overseer。

## 角色规则

- `planner` 负责维护规格、拆分任务、定义验收标准，并识别测试方向。
- `executor` 在实现前必须审查 planner 输出，挑战不合理或高风险步骤，记录采纳和拒绝的反驳意见，然后再实现。
- `tester` 根据规格和测试计划独立验证。executor 不能成为自己实现内容的最终验收者。
- `overseer` 在里程碑完成时运行，检查文档、测试和代码是否保持同步。

## 职责分离规则

同一个智能体不得同时实现生产代码并声明该代码最终验收通过。

## Overseer 日志要求

overseer 必须记录：

- 缺失或过期的规格。
- 缺失或过期的测试。
- 偏离项目目标的实现。
- 智能体重试次数过多。
- 因缺少技能或工具导致功能受阻。
- 失败命令和恢复动作。

## 必需开发流程

1. 更新或确认相关规格。
2. 编写失败测试。
3. 实现满足测试的最小代码。
4. 运行单元测试、覆盖率、构建或编译检查，以及相关 E2E 测试。
5. 在关闭大型里程碑前记录 tester 和 overseer 的审查结果。
